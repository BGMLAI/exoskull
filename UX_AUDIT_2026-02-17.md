# ExoSkull UX Audit Report
**Data:** 2026-02-17 | **Wersja:** v5.1 | **URL:** https://exoskull.xyz

---

## Executive Summary

ExoSkull to zaawansowana platforma ("Adaptive Life Operating System") z imponujacym stackiem technicznym (3D cockpit, voice pipeline, 3 motywy, 184 komponenty). Jednak **publiczne strony (landing, login) maja krytyczne problemy UX ktore zabijaja konwersje**. Dashboard i admin panel sa solidne architektonicznie, ale wymagaja polish passa.

**Ocena ogolna: 5.5/10** (silna architektura, slaba konwersja)

---

## P0 - KRYTYCZNE (napraw natychmiast)

### 1. Jezyk mieszany EN/PL na landing page
- **Problem:** Headline "Your Second Brain. Built For You. By AI." po angielsku, reszta strony po polsku. `lang="pl_PL"` w HTML.
- **Wplyw:** Dezorientuje odwiedzajacych, niszczy zaufanie. Wyglada na niedokonczony produkt.
- **Fix:** Przetlumacz headline na PL **lub** dodaj language switcher. Jedna strona = jeden jezyk.
- **Plik:** `app/(public)/page.tsx` (hero section)

### 2. Login + Rejestracja na tej samej stronie BEZ tabow
- **Problem:** Formularz logowania i rejestracji sa stacked vertycalnie. Na mobile formularz rejestracji jest daleko pod foldem.
- **Wplyw:** Nowi uzytkownicy nie znajduja rejestracji. Konwersja mobile drastycznie spada.
- **Fix:** Dodaj Tabs component (masz Radix UI Tabs) — "Zaloguj sie" | "Stworz konto". Default tab = Signup dla /login?tab=signup, Login dla /login.
- **Plik:** `app/login/page.tsx`
- **Screenshot:** `04-login-desktop-fold.png`, `08-login-mobile-fold.png`

### 3. Brak "Zapomniales hasla?"
- **Problem:** Zero opcji reset hasla na stronie logowania. Uzytkownik ktory zapomnial hasla jest zablokowany.
- **Wplyw:** Blokuje powracajacych userow. Frustracja → porzucenie.
- **Fix:** Dodaj link "Zapomniales hasla?" pod polem hasla + stworz `/reset-password` page (Supabase Auth ma wbudowany reset flow).
- **Plik:** `app/login/page.tsx`

### 4. Wyswietlanie niskich liczb social proof
- **Problem:** "4 uzytkownikow / 79 rozmow / 2 akcji AI" — live stats na landing page. Cztery osoby to anti-social-proof.
- **Wplyw:** Aktywnie zniecheca do rejestracji. Lepiej nic nie pokazywac niz male liczby.
- **Fix:** Ukryj sekcje stats do momentu gdy liczby beda imponujace (np. 100+ userow). Albo pokaz inne metryki: "56 narzedzi AI", "24/7 voice assistant", "3 motywy".
- **Plik:** `app/(public)/page.tsx` (stats section)

## P1 - WAZNE (napraw w tym tygodniu)

### 5. Brak informacji o cenach
- **Problem:** Nigdzie na stronie nie ma informacji czy ExoSkull jest darmowy, freemium, czy platny.
- **Wplyw:** Uzytkownik nie wie na co sie zapisuje. Powoduje wahanie przed kliknieciem CTA.
- **Fix:** Dodaj sekcje pricing na landing page (nawet jesli jest "Free during beta") lub jasna informacje przy CTA: "Darmowe. Bez karty kredytowej."
- **Plik:** `app/(public)/page.tsx`

### 6. Brak nawigacji na landing page
- **Problem:** Header ma tylko logo + "Zaloguj sie". Brak linkow do sekcji (Features, How it works, Testimonials).
- **Wplyw:** Uzytkownik musi scrollowac na slepo. Nie moze szybko znalezc informacji.
- **Fix:** Dodaj anchor links w headerze: "Funkcje | Jak to dziala | Opinie | Zacznij"
- **Plik:** `app/(public)/page.tsx` lub layout komponent

### 7. Testimoniale wygladaja na sfabrykowane
- **Problem:** "Michal K. (Programista)", "Anna W. (Project Manager)" — generic imiona, brak zdjec, brak linkow.
- **Wplyw:** Zamiast budowac zaufanie, podwazaja wiarygodnosc.
- **Fix:** Dodaj avatary (chocby inicjaly w kolku), pelniejsze opisy, lub usun sekcje do momentu zdobycia prawdziwych testimoniali.
- **Plik:** `app/(public)/page.tsx` (testimonials section)

### 8. Brak walidacji formularza w real-time
- **Problem:** Brak widocznych wymagan hasla (min. 6 znakow jest w HTML ale nie widac na UI). Brak walidacji email. Brak password visibility toggle.
- **Wplyw:** Uzytkownik dowiaduje sie o bledach dopiero po kliknieciu Submit.
- **Fix:** Dodaj inline validation: requirements pod polem hasla, email format check, eye icon toggle.
- **Plik:** `app/login/page.tsx`

### 9. Hero text za duzy na mobile
- **Problem:** "Your Second Brain. Built For You. By AI." zajmuje prawie caly pierwszy viewport na mobile. CTA jest pod foldem.
- **Wplyw:** Uzytkownik widzi headline ale musi scrollowac by znalezc przycisk akcji.
- **Fix:** Zmniejsz font-size na mobile (text-3xl zamiast text-5xl), lub skroc headline.
- **Screenshot:** `06-landing-mobile-fold.png`

### 10. Brak fallbacku dla 3D scene (WebGL)
- **Problem:** Dashboard uzywa Three.js/R3F. Komentarze w kodzie wspominaja "mobile/no-WebGL fallback" ale nie zaimplementowano.
- **Wplyw:** Na starszych urzadzeniach/przegladarkach dashboard moze byc pusty lub crashowac.
- **Fix:** Dodaj detekcje WebGL + static fallback (np. gradient bg + 2D layout).
- **Plik:** `components/dashboard/CyberpunkDashboard.tsx`

## P2 - ULEPSZENIA (plan na sprint)

### 11. Polityka prywatnosci / Regulamin — sciana tekstu
- **Problem:** Dlugi tekst prawny bez spisu tresci, kolapsujacych sekcji, minimalne formatowanie.
- **Fix:** Dodaj ToC z anchor links, kolapsujace sekcje, lepszy kontrast tekstu.
- **Pliki:** `app/privacy/page.tsx`, `app/terms/page.tsx`

### 12. Brak skip-to-main-content
- **Problem:** Landing page nie ma linku skip-to-content dla uzytkownikow klawiatury/screen readera.
- **Fix:** Dodaj `<a href="#main" className="sr-only focus:not-sr-only">Przejdz do tresci</a>`

### 13. Footer minimalny
- **Problem:** Tylko "Polityka Prywatnosci" i "Regulamin". Brak social links, kontaktu, opisu firmy.
- **Fix:** Dodaj social icons, email kontaktowy, krotki opis ExoSkull.

### 14. Cockpit HUD — drawer min height za maly
- **Problem:** `DRAWER_MIN = 48px` — chat jest prawie niewidoczny po zwinięciu.
- **Fix:** Zwieksz min height lub dodaj wyrazny expand indicator.
- **Plik:** `components/cockpit/CockpitHUDShell.tsx`

### 15. Admin pages — brak granularnych error boundaries
- **Problem:** Admin sekcje uzywaja bezposrednich `/api/admin/` fetch bez indywidualnych error boundaries.
- **Fix:** Wrap kazda admin sekcje w ErrorBoundary z retry.

### 16. Kontrast kolorow w Dark Ops theme
- **Problem:** Glowy i neonowe kolory moga miec niski kontrast WCAG na ciemnym tle.
- **Fix:** Audit WCAG AA compliance, popraw foreground na glow elementach.

### 17. Pusty katalog `/components/chat/`
- **Problem:** Katalog istnieje ale jest pusty — legacy artifact.
- **Fix:** Usun lub dodaj README ze wskaznikiem do nowej lokalizacji.
